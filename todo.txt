     AI & Dev News Dashboard GitHub 자동 배포 설정 할 일 목록 (TO-DO)

매일 아침 9시 자동으로 뉴스를 수집하고 블로그를 업데이트하기 위해 
사용자가 직접 진행해야 할 작업 목록입니다. 순서대로 진행해 주세요.

-------------------------------------------------------------------
1단계: GitHub에 소스 코드 올리기 (Git Push)
-------------------------------------------------------------------
로컬에 있는 블로그 프로젝트 폴더 전체를 본인의 GitHub 저장소에 업로드합니다.

* 실행 방법 (터미널/CMD에서 실행):
  1. 블로그 루트 폴더 (d:\15. seminor\blog)로 이동합니다.
     $ cd "d:\15. seminor\blog"

  2. 깃 저장소를 초기화합니다.
     $ git init

  3. 파일들을 장바구니에 담고 커밋합니다.
     $ git add .
     $ git commit -m "feat: 블로그 및 GitHub 자동화 스크래퍼 추가"

  4. 본인의 GitHub에 새 저장소(Repository)를 만들고 생성된 주소를 연결합니다.
     $ git branch -M main
     $ git remote add origin https://github.com/사용자아이디/저장소이름.git

  5. 코드를 푸시합니다.
     $ git push -u origin main

-------------------------------------------------------------------
2단계: GitHub Actions 권한 설정 (Workflow Permissions)
-------------------------------------------------------------------
GitHub Actions 가상 컴퓨터가 크롤링 결과를 깃허브에 다시 기록(Write)할 수 있게 권한을 줍니다.

  1. 웹 브라우저로 깃허브 저장소(Repository)에 접속합니다.
  2. 상단 메뉴에서 [Settings] 탭을 클릭합니다.
  3. 왼쪽 사이드바 메뉴 중 [Actions] -> [General] 을 클릭합니다.
  4. 맨 아래로 스크롤하여 [Workflow permissions] 항목으로 이동합니다.
  5. [Read and write permissions] 옵션을 체크(선택)합니다.
  6. 바로 아래 [Save] 버튼을 눌러 저장합니다.

-------------------------------------------------------------------
3단계: GitHub Pages 활성화 (블로그 웹사이트 배포)
-------------------------------------------------------------------
스크랩된 결과를 누구나 볼 수 있는 웹 페이지로 띄우는 호스팅 설정입니다.

  1. 깃허브 저장소의 [Settings] 탭으로 이동합니다.
  2. 왼쪽 사이드바 메뉴 중 [Pages] 를 클릭합니다.
  3. "Build and deployment" 섹션의 Source가 [Deploy from a branch]로 되어 있는지 확인합니다.
  4. Branch를 [main] (또는 master)로 변경하고, 우측의 폴더는 [/ (root)]로 둔 채 [Save]를 누릅니다.
  5. 약 1~2분 뒤 페이지 새로고침을 하면 상단에 완성된 나만의 블로그 주소가 나타납니다.
     (예: https://사용자아이디.github.io/저장소이름/)

-------------------------------------------------------------------
4단계: 자동 스크랩 테스트 해보기 (Workflow Dispatch)
-------------------------------------------------------------------
다음날 아침 9시까지 기다리지 않고, 스크래퍼가 깃허브에서 정상 작동하는지 수동으로 실행해 봅니다.

  1. 깃허브 저장소 상단 메뉴의 [Actions] 탭을 클릭합니다.
  2. 왼쪽의 Workflows 목록에서 [Daily AI & Dev News Scraper]를 클릭합니다.
  3. 우측에 나타나는 [Run workflow] 버튼을 누르고 초록색 [Run workflow]를 클릭합니다.
  4. 스크립트 실행이 완료된 후 [Code] 탭에서 data/news.json 파일이 갱신되었는지 확인합니다.
